# Применение графовых нейронных сетей в рекомендательных системах

Репозиторий содержит материалы выпускной квалификационной работы, посвящённой исследованию графовых нейронных сетей и гибридных моделей для top-K рекомендации фильмов по истории явных пользовательских оценок.

Основная идея проекта состоит в том, чтобы использовать численное значение рейтинга не только как бинарный отклик, но и как дополнительный сигнал в структуре user-item графа. В работе рассматриваются способы учёта рейтинга как типа ребра, как веса связи, а также в составе гибридных моделей, объединяющих последовательные и графовые представления.

## Аннотация

В рекомендательных системах взаимодействия пользователя с объектом часто сводятся к бинарному факту: пользователь посмотрел фильм, поставил лайк или совершил покупку. Однако в случае явных оценок доступен более информативный сигнал: пользователь не просто взаимодействовал с объектом, а выразил степень предпочтения числом на рейтинговой шкале.

В данном проекте исследуется задача рекомендации фильмов по истории явных пользовательских оценок. Эксперименты проводятся на подготовленном датасете на основе MovieLens 25M, дополненном признаками фильмов из MovieLens Genome и IMDb. Для оценки используются warm-start, cold-user, cold-item и both-cold сценарии.

В качестве моделей рассматриваются матричная факторизация, двубашенная модель SASRec-вида, GCMC / rating-as-relation, NGCF-подобная модель с учётом рейтинговых весов, а также ансамбль двубашенной и графовой моделей. Качество сравнивается по метрикам ранжирования Precision@K, Recall@K, HitRate@K, MRR@K, MAP@K и NDCG@K.

## Цель работы

Цель работы — исследовать, насколько полезен явный численный рейтинг в графовых и гибридных рекомендательных моделях для задачи top-K рекомендации фильмов.

Для достижения цели рассматриваются следующие задачи:

- подготовка датасета MovieLens-based с временным разбиением и признаками фильмов;
- построение warm-start и cold-start сценариев оценки;
- реализация baseline-моделей для top-K рекомендации;
- исследование способов включения численного рейтинга в графовые модели;
- сравнение графовых, неграфовых и гибридных подходов;
- анализ качества моделей по метрикам ранжирования и времени работы.

## Датасет

В работе используется подготовленный датасет на основе MovieLens 25M. Данные дополнены контентными признаками фильмов из MovieLens Genome и IMDb.

В датасете сохранены:

- пользовательские оценки фильмов;
- временные метки взаимодействий;
- признаки фильмов;
- warm-start и cold-start разбиения;
- служебные отображения идентификаторов в модельные индексы.

Разбиение выполняется по времени отдельно для каждого пользователя. Validation и test формируются из последних пользовательских взаимодействий, при этом исходные численные рейтинги сохраняются в таблицах и интерпретируются уже на этапе конкретного эксперимента.

Основные сценарии оценки:

| Сценарий | Описание |
|---|---|
| `warm` | Пользователь и фильм присутствуют в обучающем графе |
| `cold_user` | Новый пользователь имеет только support-историю |
| `cold_item` | Новый фильм отсутствует в train, но имеет признаки |
| `both_cold` | Новый пользователь и новый фильм одновременно |

Подробное описание подготовки данных находится в файле [`docs/dataset_description.md`](docs/dataset_description.md).

## Рассматриваемые модели

В проекте сравниваются следующие подходы.

| Модель | Краткое описание |
|---|---|
| `MF` | Матричная факторизация как базовый collaborative filtering baseline |
| `Two-Tower / SASRec-like` | Двубашенная модель, строящая представление пользователя по истории оценок |
| `GCMC / rating-as-relation` | Графовая модель, где разные значения рейтинга задают разные типы рёбер |
| `NGCF rating-weighted` | NGCF-подобная модель, где рейтинг используется как вес связи |
| `Ensemble TT-GCMC` | Гибридная модель, объединяющая представления двубашенной и графовой моделей |

Схемы архитектур расположены в папке [`architectures/`](architectures/).  
Описание методов приведено в файле [`docs/methods.md`](docs/methods.md).

## Метрики

Для оценки качества рекомендаций используются top-K метрики:

- `Precision@K`;
- `Recall@K`;
- `HitRate@K`;
- `MRR@K`;
- `MAP@K`;
- `NDCG@K`.

Основные значения `K` выбираются в соответствии с экспериментальной постановкой. Оценка проводится отдельно для warm-start и cold-start сценариев.

## Описание разделов

### `architectures/`

Папка содержит схемы архитектур и графового распространения сообщений:

- `SASRec_like.png` — схема двубашенной модели SASRec-вида;
- `GCMC.png` — схема GCMC / rating-as-relation подхода;
- `NGCF.png` — схема NGCF-подобной модели;
- `message_pass.png` — иллюстрация message passing в user-item графе.

### `configs/`

Папка содержит конфигурации основных запусков:

- `mf.yaml`;
- `two_tower_sasrec.yaml`;
- `gcmc_rating_relation.yaml`;
- `ngcf_rating_weighted.yaml`;
- `ensemble_two_tower_gcmc.yaml`.

Конфиги фиксируют основные параметры моделей и обучения: размерности эмбеддингов, число слоёв, learning rate, batch size, параметры регуляризации и настройки оценки.

### `docs/`

Папка содержит текстовые материалы проекта:

- [`dataset_description.md`](docs/dataset_description.md) — описание датасета и схемы разбиения;
- [`methods.md`](docs/methods.md) — описание рассматриваемых моделей;
- [`experiments.md`](docs/experiments.md) — описание экспериментальной постановки;
- [`repository_notes.md`](docs/repository_notes.md) — дополнительные пояснения к структуре репозитория;
- `thesis/` — текст ВКР;
- `presentation/` — презентация к защите.

### `notebooks/`

Папка содержит Jupyter-ноутбуки с основным кодом исследования:

- `01_dataset_eda.ipynb` — анализ датасета и проверка подготовленных таблиц;
- `02_baselines_mf_two_tower_gcmc.ipynb` — обучение и сравнение базовых моделей.

### `results/`

Папка содержит итоговые результаты экспериментов:

- `results/tables/` — таблицы с метриками;
- `results/figures/` — графики и визуализации;
- `results/README.md` — описание файлов с результатами.
