# Применение графовых нейронных сетей в рекомендательных системах

Репозиторий содержит материалы выпускной квалификационной работы по теме  
**«Применение графовых нейронных сетей в рекомендательных системах»**.

Работа посвящена задаче рекомендации фильмов по истории явных пользовательских оценок. Основной акцент сделан на исследовании графовых рекомендательных моделей, способов учёта численного рейтинга в графе «пользователь - фильм» и гибридных архитектур, объединяющих графовые представления с двубашенной моделью.

## Аннотация

В рекомендательных системах взаимодействие пользователя с объектом часто рассматривается как бинарный сигнал: пользователь либо взаимодействовал с объектом, либо нет. В случае явных пользовательских оценок доступен более информативный сигнал — численный рейтинг, отражающий степень предпочтения пользователя.

В работе исследуется, как использовать явные оценки в задаче построения первых \(K\) рекомендаций фильмов. Рассматриваются несколько способов учёта рейтинга: как бинарной релевантности для метрик ранжирования, как типа отношения в GCMC-подобной графовой модели, как веса ребра в NGCF-подобной модели, а также как дополнительного сигнала в гибридных моделях.

Эксперименты проводятся на подготовленном датасете на основе MovieLens с дополнительными признаками фильмов. Для оценки используются сценарии тёплого старта, холодного пользователя, холодного фильма и одновременного холодного пользователя и фильма. В качестве моделей сравниваются матричная факторизация, двубашенная модель SASRec-вида, GCMC, NGCF-подобная модель с рейтинговыми весами и ансамбль двубашенной и графовой моделей.

## Цель работы

Цель работы — исследовать применение графовых нейронных сетей в рекомендательных системах фильмов и сравнить способы учёта явных пользовательских рейтингов в графовых и гибридных моделях.

Основные задачи проекта:

- подготовить датасет для рекомендации фильмов с сохранением исходных численных рейтингов;
- построить временное разбиение на обучающую, валидационную и тестовую части;
- выделить отдельные сценарии оценки для тёплого старта и холодного старта;
- реализовать и сравнить базовые модели рекомендаций;
- исследовать графовые модели, использующие рейтинг как тип отношения и как вес связи;
- построить гибридную модель, объединяющую двубашенное и графовое представления;
- сравнить качество моделей по метрикам ранжирования.

## Датасет

В проекте используется подготовленный датасет на основе MovieLens с дополнительными признаками фильмов. Взаимодействия пользователей и фильмов сохраняются вместе с исходными численными рейтингами и временными метками.

Датасет содержит:

- обучающие, валидационные и тестовые взаимодействия;
- исходные значения рейтингов;
- признаки фильмов;
- сценарии тёплого старта и холодного старта;
- служебные отображения идентификаторов пользователей и фильмов в модельные индексы.

Ключевая особенность подготовки данных состоит в том, что рейтинг не бинаризуется на этапе сборки датасета. Это позволяет задавать способ интерпретации рейтинга непосредственно в эксперименте.

Описание датасета и схемы разбиения приведено в файле [`docs/dataset_description.md`](docs/dataset_description.md).

## Сценарии оценки

| Сценарий | Описание |
|---|---|
| `warm` | Пользователи и фильмы присутствуют в обучающем графе |
| `cold_user` | Новый пользователь представлен через вспомогательную историю оценок |
| `cold_item` | Новый фильм отсутствует в обучающей части, но имеет признаки |
| `both_cold` | Пользователь и фильм одновременно отсутствуют в обучающем графе |

## Рассматриваемые модели

| Модель | Описание |
|---|---|
| `MF` | Матричная факторизация как базовая модель коллаборативной фильтрации |
| `Two-Tower / SASRec-like` | Двубашенная модель, строящая представление пользователя по истории оценок |
| `GCMC` | Графовая модель, в которой значение рейтинга задаёт тип ребра |
| `NGCF rating-weighted` | NGCF-подобная модель, в которой рейтинг используется как вес связи |
| `Ensemble Two-Tower + GCMC` | Гибридная модель, объединяющая двубашенные и графовые представления |

Схемы архитектур находятся в папке [`architectures/`](architectures/), а описание методов — в файле [`docs/methods.md`](docs/methods.md).

## Использование явных рейтингов

В проекте рассматриваются несколько способов использования рейтинга.

### Бинарная релевантность

Для стандартных метрик ранжирования фильм считается релевантным при выполнении условия:

```text
rating >= 4
```

### Рейтинг как тип ребра

В GCMC-подобной модели взаимодействие пользователя и фильма задаётся как тройка:

```text
(user, item, rating)
```

Разные значения рейтинга рассматриваются как разные типы отношений в графе «пользователь — фильм».

### Рейтинг как вес связи

В NGCF-подобной модели рейтинг используется как вес ребра при распространении сообщений по графу. Более высокая оценка соответствует более сильной связи между пользователем и фильмом.

### Гибридное представление

В гибридной модели объединяются представления, полученные двубашенной моделью и графовой моделью. Такой подход позволяет совместить описание пользовательской истории с сигналом из графа взаимодействий.

## Метрики качества

Модели оцениваются в задаче построения первых \(K\) рекомендаций по следующим метрикам:

- `Precision@K`;
- `Recall@K`;
- `HitRate@K`;
- `MRR@K`;
- `MAP@K`;
- `NDCG@K`.

Оценка проводится отдельно для сценариев тёплого старта и холодного старта.
