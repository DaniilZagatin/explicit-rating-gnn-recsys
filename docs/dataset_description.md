# Датасет

В экспериментах используется подготовленный датасет на основе MovieLens с дополнительными признаками фильмов.

Основная особенность датасета состоит в том, что исходные пользовательские оценки сохраняются в численном виде. Бинаризация не выполняется на этапе подготовки данных: решение о том, как интерпретировать рейтинг, задаётся уже в конкретном эксперименте.

## Что входит в датасет

- взаимодействия пользователей и фильмов;
- численные рейтинги;
- временные метки;
- признаки фильмов;
- метаданные пользователей и фильмов;
- mapping-файлы для модельных индексов;
- train/validation/test разбиения;
- warm-start, cold-user, cold-item и both-cold сценарии.

## Сплит по времени

Разбиение строится отдельно для каждого пользователя. Его взаимодействия сортируются по времени, после чего последние положительные оценки используются для формирования validation/test окон. При этом в оценочных окнах сохраняются все рейтинги, попавшие в соответствующий временной диапазон, а не только положительные.

Такой сплит по времени лучше соответствует задаче рекомендации: модель обучается на более ранней истории и оценивается на более поздних взаимодействиях.
