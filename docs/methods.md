# Модели

В проекте сравниваются несколько классов рекомендательных моделей. Основной интерес связан с графовыми моделями и их ансамблями с двубашенными архитектурами.

## Матричная факторизация

Базовая collaborative filtering модель. Используется как простой ориентир качества для warm-start сценария.

## Двубашенная модель SASRec-вида

Двубашенная модель строит представление пользователя и фильма в общем векторном пространстве. User tower использует историю пользователя и последовательный encoder SASRec-вида, item tower строит представление объекта по embedding и признакам фильма.

Эта модель важна как baseline, потому что она поддерживает работу с историей пользователя и признаками объектов.

## GCMC / rating-as-relation

GCMC-подобная модель рассматривает значение рейтинга как тип ребра в user-item графе. В таком подходе оценки `5`, `3` и `1` соответствуют разным отношениям между пользователем и фильмом.

## NGCF-подобная модель с весами рейтинга

В NGCF-подобной модели рейтинг используется как вес связи между пользователем и фильмом. Высокая оценка усиливает вклад соседнего узла при распространении сигнала, низкая — ослабляет.

## Гибридная модель Two-Tower + GCMC

Гибридная модель объединяет представления, полученные из двубашенной модели и графовой модели. Идея состоит в том, что две архитектуры используют разные источники сигнала:

- двубашенная модель учитывает последовательную историю пользователя и признаки фильмов;
- графовая модель использует структуру user-item взаимодействий;
- ансамбль объединяет оба вида представлений.
