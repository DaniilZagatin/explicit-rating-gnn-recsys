# Изображения проекта

В этой папке лежат схемы моделей и графики, которые используются в README, документации и материалах по экспериментам.  
Файлы разделены по смыслу: часть изображений описывает архитектуры, часть — датасет и результаты сравнения моделей.

## Схемы моделей

| Файл | Что показано |
|---|---|
| `SASRec_like.png` | Двубашенная модель SASRec-вида: пользовательское представление строится по истории, объектное представление — через embedding/encoder фильма, итоговый score считается скалярным произведением. |
| `GCMC.png` | Общая схема GCMC: user-item граф с рейтингами, графовый encoder и bilinear decoder для предсказания оценок. |
| `message_pass.png` | Иллюстрация message passing: как представление узла обновляется через соседей на разных hop-уровнях. |
| `NGCF.png` | Схема NGCF-подобной модели: propagation-слои, объединение embedding-векторов разных уровней и prediction layer. |

## Графики по данным

| Файл | Что показано |
|---|---|
| `dataset.png` | Распределение логарифма длины пользовательских историй. График используется для проверки масштаба и неоднородности пользовательской активности. |

## Графики по экспериментам

| Файл | Что показано |
|---|---|
| `all_models_warm_hr20.png` | Сравнение моделей по `Warm HR@20`. Используется как общий обзор качества в warm-start сценарии. |
| `warm_vs_cold_user_tradeoff.png` | Сравнение качества в warm-start и cold-user сценариях. График показывает, как модели ведут себя при переходе от известных пользователей к новым. |
| `quality_time_tradeoff.png` | Сравнение качества и времени оценки на 100k пользователей. Используется для анализа компромисса между качеством и вычислительной стоимостью. |
| `rating_weight_learned_function.png` | Примеры функций веса ребра `w(r)` для NGCF-подобной модели. Показывает, как значение рейтинга может влиять на силу связи в графе. |
| `validation_tt_vs_ens_gcmc.png` | Динамика качества на валидации для двубашенной модели и гибридного ансамбля `Ens-TT-GCMC-ft`. |

## Как использовать

Изображения можно вставлять в README или markdown-документы через относительные ссылки:

```markdown
![Схема GCMC](assets/GCMC.png)
```

Для файлов из `results/figures/` путь нужно заменить на соответствующий:

```markdown
![Сравнение моделей](results/figures/all_models_warm_hr20.png)
```

## Замечания

- Схемы архитектур лучше хранить в `assets/`, так как они описывают устройство моделей.
- Графики с результатами экспериментов лучше хранить в `results/figures/`.
- Имена файлов стоит оставлять короткими и стабильными, чтобы ссылки в README и документации не ломались.
